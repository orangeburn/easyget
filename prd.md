Easyget初步方案设计
本方案旨在通过 AI 代理（Agent）与自动化浏览器技术，实现招标信息的全自动化获取、过滤与深度价值评估。

1. 核心设计原则
- **结果优先**：系统核心为单栏线索列表。顶部状态栏显示当前采集策略与实时进度。
- **结构化信息**：线索列表直接展示项目名、发布公司、所在地。
- **逻辑匹配透明化**：以绿色/红色数字形式，直白展示当前线索满足/不符合的画像条件数量。
- **智能去重**：后台自动过滤抓取到的重复招标公告。
“这单值不值得投”。

2. 核心组件设计
第一部分：需求解析器 (Requirement Parser)
- **多步引导流**：
  1. **文档初始化**：提取企业能力标签。
  2. **资质核对**：结构化确认/勾选已有资质。
  3. **动态补充**：基于行业逻辑生成专用补充表单。
  4. **全网/定向配置**：设定搜索指令、域名监控及公众号名单。
- **产出**：结构化的 `Business-Constraint.json`。

第二部分：信息采集引擎 (Collection Engine)
- **三模引擎**：全网搜索（关键词驱动）、定向 URL 抓取、微信公众号（搜狗接口）。
- **调度策略**：支持设定低/中/高频扫描，平衡实时性与反爬安全性。

第三部分：线索分析器 (Clue Analyzer)
- **分析算法**：LLM 结构化提取元数据 -> 语义去重 -> 一票否决自检 -> 加权打分评分。

3. 数据流向 (Data Flow)
解析层：原始资料 -> 行业推理表单 -> 任务配置 -> 生成画像。
执行层：画像驱动 -> 混合采集 -> 指纹伪装 -> 原始库。
处理层：瀑布流数据 -> LLM 筛选 -> 一票否决 -> 评分排序。
展示层：首屏瀑布流 -> 侧滑详情 -> 专家建议。
