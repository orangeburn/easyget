AI 招标情报中枢 (AI-Tender Intelligence Hub) 初步方案设计
本方案旨在通过 AI 代理（Agent）与自动化浏览器技术，实现招标信息的全自动化获取、过滤与深度价值评估。
1. 核心组件设计
第一部分：需求解析器 (Requirement Parser)
功能描述：负责业务分析与画像定义。
交互机制：采用“苏格拉底式”引导提问。AI 不会直接要求用户填写表格，而是通过对话挖掘潜在约束（例如：从“活动场地”推导出“消防资质”、“承重限制”、“供电容量”等）。
输出产物：动态生成 Business-Constraint.json。
动态调整：支持用户随时修正画像。例如用户输入“最近我们拿到了跨省施工资质”，解析器将自动更新地理约束条件。
第二部分：信息采集引擎 (Collection Engine)
功能描述：混合模式线索采集。
搜索引擎模式：集成全网 API 进行广度扫描。
定向搜集模式：针对特定 URL 或明确公众号名进行深度抓取。
微信文章模式：针对微信生态进行专项攻坚（存量挖掘与实时推送）。
第三部分：反爬虫策略引擎 (Anti-Crawling Engine)
功能描述：隐形化采集保障。
环境伪装：集成 playwright-stealth 抹除浏览器特征。
IP 调度：对接住宅代理，实现 IP 轮换，防止触发频率限制。
第四部分：线索分析器 (Clue Analyzer)
功能描述：线索清洗、评估与量化。
动态清洗：根据画像提取元数据（金额、地点、截止期）。
评估体系：实现一票否决逻辑与加权打分排序。
2. 数据流向图 (Data Flow)
解析层：用户输入 -> 苏格拉底提问 -> 生成业务画像。
执行层：画像驱动三模引擎同步采集。
防御层：所有请求经过指纹伪装与 IP 轮换。
处理层：原始数据 -> LLM 结构化 -> 语义去重 -> 匹配评分。
